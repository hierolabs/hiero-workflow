## 1. 개념도: 가격이 흐르는 구조

```
                          가격 흐름 (Price Flow)

 [PriceLabs]              [Hostex]              [Airbnb / OTA]
 시장분석 엔진              채널 허브               판매 채널
 ┌──────────┐          ┌──────────┐          ┌──────────────┐
 │ 경쟁숙소  │    (1)   │          │    (3)   │              │
 │ 수요예측  │────────→ │ 가격     │────────→ │  게스트에게   │
 │ AI 추천가 │ 자동가격 │ 동기화   │  푸시    │  노출/판매    │
 └──────────┘          └────┬─────┘          └──────┬───────┘
                            │                       │
                       (2-1)│DB 저장            (3-1)│판매기록
                            │                       │웹훅
                            ▼                       ▼
                     ┌─────────────────────────────────┐
                     │           HIERO                  │
                     │      데이터 허브 + 의사결정       │
                     │                                  │
                     │  listing_calendars  (가격 캐시)   │
                     │  price_labs_prices  (추천가)      │
                     │  reservations       (판매기록)    │
                     │  hostex_transactions(정산기록)    │
                     │                                  │
                     │  ┌─ 가격 캘린더 (비교/편집)       │
                     │  ├─ 이상치 알림 (추천가 vs 현재)  │
                     │  └─ 수동 개입 → Hostex 푸시  (4)  │
                     └─────────────────────────────────┘
```

### 번호별 설명

| # | 흐름 | 설명 |
|---|------|------|
| (1) | PriceLabs → Hostex | PriceLabs가 시장 데이터 분석 후 자동 가격 조정. Hostex 채널 매니저에 자동 전송 |
| (2-1) | Hostex → HIERO DB | HIERO가 Hostex 캘린더 API로 가격 데이터 수집, listing_calendars 테이블에 캐시 |
| (3) | Hostex → OTA | Hostex가 Airbnb/Agoda 등 OTA에 가격 푸시. 게스트에게 노출 |
| (3-1) | OTA → Hostex → HIERO | 예약 발생 시 웹훅으로 HIERO에 판매 기록 이관 (reservations, hostex_transactions) |
| (4) | HIERO → Hostex | 운영자가 HIERO 가격 캘린더에서 수동 가격 변경 시 Hostex API로 푸시 |

## 2. 각 시스템의 역할

### PriceLabs (가격 결정)
- **역할**: 시장 분석 + AI 기반 가격 추천
- **데이터**: 경쟁 숙소 가격, 지역 수요, 시즌, 이벤트, 점유율
- **출력**: 날짜별 추천 가격 → Hostex로 자동 전송
- **HIERO 연동**: API로 추천가/수요 지표 수집 → price_labs_prices 테이블

### Hostex (채널 허브)
- **역할**: 멀티 OTA 가격/가용성 동기화
- **기능**: PriceLabs 가격 수신 → Airbnb/Agoda/Booking 동시 반영
- **HIERO 연동**: 
  - 읽기: GET /listings/calendar → 현재 가격/가용성 조회
  - 쓰기: POST /listings/prices, /listings/restrictions → 가격/최소숙박/차단 변경

### HIERO (데이터 허브 + 의사결정)
- **역할**: 관찰 + 비교 + 수동 개입
- **하는 것**:
  - PriceLabs 추천가 vs Hostex 현재가 비교
  - 이상치 감지 (±20% 이상 차이)
  - 특정 날짜 가격 직접 변경 (Hostex 경유)
  - 변경 이력 기록 (activity_logs)
- **안 하는 것**:
  - 자동 가격 결정 (PriceLabs 담당)
  - OTA 직접 연동 (Hostex 담당)

## 3. DB 테이블 구조

### listing_calendars (가격 캐시)
| 컬럼 | 설명 |
|------|------|
| property_id | 숙소 ID |
| date | 날짜 (YYYY-MM-DD) |
| price | 1박 가격 (원) |
| min_stay | 최소 숙박 (박) |
| available | 예약 가능 여부 |
| closed_on_arrival | 체크인 차단 |
| closed_on_departure | 체크아웃 차단 |
| synced_at | 마지막 동기화 시각 |

- **데이터량**: 53개 숙소 x 120일 = ~6,500건
- **동기화 주기**: 수동 트리거 (POST /admin/pricing/sync)
- **범위**: 30일 전 ~ 90일 후

### price_labs_prices (추천가)
| 컬럼 | 설명 |
|------|------|
| listing_id | PriceLabs 리스팅 ID |
| date | 날짜 |
| price | PriceLabs 추천 가격 |
| min_stay | 추천 최소 숙박 |
| booking_status | 예약 상태 |
| demand_color | 수요 색상 코드 |
| demand_desc | 수요 설명 (높음/보통/낮음) |

## 4. 가격 편집 프로세스

### 4-1. 일괄 가격 변경

```
가격 캘린더 접속 (/price-calendar)
    │
    ├─ 숙소 행에서 날짜 셀 드래그 선택
    │   (같은 숙소의 연속된 날짜)
    │
    ├─ 모달 팝업
    │   ├ 가격 변경 (고정금액 or % 조정)
    │   ├ 최소 숙박 변경 (1~30박)
    │   └ 차단/해제
    │
    ├─ "Hostex에 반영" 클릭
    │   ├ → HIERO 백엔드 API 호출
    │   ├ → Hostex API 호출 (가격 푸시)
    │   ├ → DB 캐시 업데이트
    │   └ → activity_logs에 변경 이력 기록
    │
    └─ 화면 자동 새로고침
```

### 4-2. API 엔드포인트

| 메서드 | 경로 | 기능 |
|--------|------|------|
| GET | /admin/pricing/calendar?start=&end= | 가격 조회 (DB 캐시) |
| PUT | /admin/pricing/price | 가격 변경 → Hostex 푸시 |
| PUT | /admin/pricing/restrictions | 최소숙박 변경 → Hostex 푸시 |
| PUT | /admin/pricing/availability | 차단/해제 → Hostex 푸시 |
| POST | /admin/pricing/sync | Hostex → DB 전체 동기화 |
| GET | /admin/pricing/links | 숙소별 외부 플랫폼 링크 |
| GET | /admin/pricelabs/compare | Hostex vs PriceLabs 비교 |

## 6. 앞으로 구현할 것

| 우선순위 | 기능 | 설명 |
|----------|------|------|
| 1 | 가격 변경 이력 | 누가, 언제, 어떤 숙소, 얼마→얼마 변경했는지 추적 |
| 2 | 이상치 자동 알림 | PriceLabs 추천가 대비 ±20% 이상 차이 시 알림 |
| 3 | 기본가/주말가 설정 | 숙소별 기본 가격 규칙 (평일/금토) |
| 4 | 채널별 가격 비율 | Airbnb 100%, Agoda 90% 등 채널별 차등 |
| 5 | 가격 동기화 자동화 | 현재 수동 → 주기적 자동 동기화 (cron) |

## 7. 주의사항

- **HIERO는 가격을 결정하지 않는다** — PriceLabs가 결정, HIERO는 모니터링 + 수동 개입
- **Hostex가 유일한 쓰기 경로** — HIERO → Hostex → OTA. OTA 직접 연동 없음
- **DB는 캐시** — listing_calendars는 Hostex 데이터의 사본. 진짜 가격은 Hostex에 있음
- **Airbnb만 가격 제어** — Agoda/Booking 가격은 Hostex 채널 설정에서 Airbnb 대비 비율로 자동 조정
