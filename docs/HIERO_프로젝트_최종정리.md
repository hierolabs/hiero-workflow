# 📘 HIERO 프로젝트 최종 정리
## 100채 → 300채 확장 프로젝트 (최종 버전)

**상태:** 백엔드 개발 진행 중

**목표:** 월 3.5억원 (6개월)

## 🎯 프로젝트 개요

### 현재 상황
```
운영 숙소: 100채
월 매출: 2억원
팀 규모: 5명

문제:
- 팀 협업 미흡
- 수동 작업 많음
- 자동화 부족
```

### 해결책
```
1️⃣ HIERO 운영 OS 설계
   └─ 5개 팀 구조 (27명 규모)

2️⃣ 워크플로우 시스템 개발
   └─ 자동 일 흐름 (4주)

3️⃣ 200채 마케팅
   └─ 5개 채널 병렬 운영

4️⃣ AI 자동화
   └─ Multi-AI OS 연동
```

### 최종 목표

## 💡 개발 팁

### MySQL 명령어
```bash
# 접속
mysql -u root -p

# 데이터베이스 확인
SHOW DATABASES;

# 테이블 확인
USE hiero_workflow;
SHOW TABLES;
DESC tasks;

# 데이터 확인
SELECT * FROM tasks;
SELECT * FROM task_history;

# 나가기
\q
```

### Express 개발 팁
```bash
# 개발 모드 실행
npm run dev

# 자동 재시작됨 (nodemon 사용)

# 정지하려면
Ctrl + C
```

### API 테스트 (curl)
```bash
# GET
curl http://localhost:3001/health

# POST
curl -X POST http://localhost:3001/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"title":"테스트","team":"sales","description":"테스트 설명"}'

# PATCH
curl -X PATCH http://localhost:3001/api/tasks/1/status \
  -H "Content-Type: application/json" \
  -d '{"status":"IN_PROGRESS"}'
```
