# 🔄 HIERO Workflow System (팀 자동화 도구)
## Asana처럼 팀 간 일이 자동으로 흐르는 시스템

---

## 🎯 핵심 개념

```
영업팀의 한 액션 → 여러 팀의 자동 태스크 생성

예시:
"숙소 A 계약 체결" ✅
  ↓ (자동)
청소팀에 태스크: "숙소 A 초기 세팅" 생성
  ↓ (청소팀 완료)
데이터팀에 태스크: "HIERO 등록" 생성
  ↓ (데이터팀 완료)
CEO에게 알림: "🎉 새 숙소 LIVE"
```

---

## 🏗️ 시스템 구조

```
┌─────────────────────────────────────────┐
│        Workflow Dashboard (웹)           │ ← 팀원이 보는 화면
│  태스크 보드 + 진행상황 + 자동 알림      │
└─────────────────────────────────────────┘
         ↓ (React 프론트)
┌─────────────────────────────────────────┐
│      Workflow Backend API                │ ← 데이터 처리
│  POST /tasks | GET /dashboard            │
│  PATCH /task/:id/status (상태 변경)      │
└─────────────────────────────────────────┘
         ↓ (Node.js/Express)
┌─────────────────────────────────────────┐
│    Automation Engine (자동화 로직)        │
│  태스크 생성 규칙 | 상태 변경 트리거      │
│  Claude API 호출 (자동 메시지 생성)      │
└─────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────┐
│        Database (PostgreSQL)              │
│  tasks | workflows | team_members        │
│  task_history | notifications            │
└─────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────┐
│     HIERO 플랫폼과 동기화                │
│  숙소 추가 → DB 반영                    │
│  청소 상태 → 예약 상태 변경             │
└─────────────────────────────────────────┘
```

## 🔧 시스템 기술 스택

### 프론트엔드
```
React + TypeScript
├─ 페이지:
│  ├─ Dashboard (전체 현황)
│  ├─ TaskBoard (팀별 - 드래그앤드롭)
│  ├─ TaskDetail (상세 정보)
│  ├─ Analytics (성과 분석)
│  └─ Settings (설정)
├─ 라이브러리:
│  ├─ react-beautiful-dnd (드래그)
│  ├─ recharts (차트)
│  ├─ zustand (상태관리)
└─ 배포: Vercel

디자인:
- Tailwind CSS
- 모바일 반응형 필수
```

### 백엔드
```
Node.js + Express (또는 NestJS)
├─ API 엔드포인트:
│  ├─ POST /api/tasks (생성)
│  ├─ GET /api/tasks (조회)
│  ├─ PATCH /api/tasks/:id/status (상태변경)
│  ├─ GET /api/dashboard (대시보드)
│  └─ POST /api/webhooks (자동화)
├─ 자동화:
│  ├─ 태스크 자동 생성 (영업팀 상태변경 감지)
│  ├─ 알림 발송 (Slack/Email)
│  └─ Claude API 호출 (메시지 생성)
└─ 배포: Replit 또는 AWS

WebSocket:
- 실시간 알림
- 라이브 업데이트
```

### 데이터베이스
```
PostgreSQL
├─ 테이블:
│  ├─ tasks (id, title, status, team, created_by, created_at)
│  ├─ task_subtasks (부작업)
│  ├─ task_comments (댓글)
│  ├─ team_members (팀원 정보)
│  ├─ workflows (자동화 규칙)
│  ├─ notifications (알림)
│  └─ task_history (이력)
└─ 백업: 매일 자정 + AWS S3
```

### 자동화 (Multi-AI OS 연결)
```
Webhook 기반:

1. 태스크 생성 규칙
   영업팀: CONTRACT_SIGNED
   → 청소팀: SETUP_PENDING 자동 생성
   → 데이터팀: INFO_INPUT_PENDING 자동 생성

2. Claude API 호출
   청소팀이 SETUP_COMPLETE
   → Claude가 완료 메시지 생성
   → Slack에 자동 발송

3. 알림 발송
   지연 감지 → Slack 알림
   완료 → Email 알림
   CEO 대시보드 → 실시간 업데이트
```

## 💡 핵심 이점

```
Before (지금):
- 영업팀이 계약 체결 → 카톡으로 청소팀에 알림
- 청소팀이 청소함 → 엑셀에 기록
- 데이터팀이 수동으로 HIERO 등록
- CEO가 따로 수익 계산

문제:
- 정보 누락
- 타이밍 지연
- 의사소통 비효율
- 데이터 오류

After (이 시스템):
- 영업팀이 상태 변경 → 자동으로 다음 팀 알림
- 청소팀 → 자동으로 전후 사진 데이터팀에 전달
- 데이터팀 → 자동으로 HIERO 등록
- CEO → 매분 실시간 대시보드 확인

이점:
✅ 지연 0 (실시간)
✅ 오류 0 (자동)
✅ 정보 투명 (중앙화)
✅ 의사결정 빠름 (데이터 기반)
```
