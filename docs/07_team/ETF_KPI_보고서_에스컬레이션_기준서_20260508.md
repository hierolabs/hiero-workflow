# ETF KPI · 보고서 · 에스컬레이션 기준서

> 작성: 2026년 5월 8일
> 목적: CEO/CTO/CFO의 월간 보고서 내용, KPI 목표, 하부 조직 에스컬레이션 기준을 확정한다.

---

## 1. CEO 월간 경영 보고서

### 보고서 내용

```txt
1. 경영 요약 (한 달 핵심 3줄)
2. 포트폴리오 현황
   - 총 숙소 수, 운영중/중단/신규
   - 권역별 분포
   - 계약 만료 예정
3. 예약·매출 KPI
   - 가동률, ADR, RevPAR
   - 채널별 매출 비중 (Airbnb/Booking/Agoda/자리톡/삼삼엠투)
   - reservation_date 기준 월 매출
4. 운영 효율
   - 이슈 총 건수 / 해결률 / 평균 해결시간
   - 반복 이슈 Top 5
   - 청소 완료율 / 사진 누락률
5. 팀 성과
   - 팀원별 이슈 처리, 에스컬레이션, 활동량
   - 병목 구간 분석
6. 성장 (Growth)
   - 신규 리드 수 / 전환율
   - 랜딩페이지 방문수
   - 제안서 발송 건수
7. 의사결정 기록
   - Founder Brief 올라간 건수
   - 승인/반려/돌려보내기 이력
8. 다음 달 액션 플랜
```

### CEO KPI (월간 달성 목표)

| KPI | 목표 | 측정 기준 |
|-----|------|---------|
| 전체 가동률 | ≥ 75% | reservations / (properties × days) |
| ADR | ≥ 65,000원 | 총매출 / 총 숙박일수 |
| 이슈 해결률 | ≥ 85% | resolved / (open + resolved) 월간 |
| 평균 해결시간 | ≤ 12시간 | AVG(resolved_at - created_at) |
| 팀 병목 건수 | ≤ 3건 | P0/P1 미해결 24시간 초과 |
| 신규 리드 전환율 | ≥ 20% | converted_leads / total_leads |
| Founder 에스컬레이트 | ≤ 5건/월 | escalation_level=founder from CEO |
| 청소 완료율 | ≥ 95% | completed / scheduled |

## 2. CFO 월간 재무 보고서 + 세무 자료

### 보고서 내용

```txt
1. 재무 요약 (한 달 핵심 3줄)
2. 매출 분석
   - reservation_date 기준 발생 매출
   - deposit_date 기준 입금 예정
   - transaction_at 기준 실제 입금
   - 채널별 매출 (과세/면세 구분)
3. 비용 분석
   - cost_allocations 기준 월별 배분
   - 항목별: 청소비, 월세, 관리비, 수리비, 소모품, 인건비
   - 전월 대비 증감
4. 숙소별 P&L
   - 수익률 TOP 5 / BOTTOM 5
   - 적자 숙소 원인 분석
   - 권역별 수익률
5. 현금 흐름
   - 입금 예정 vs 실제 입금 차이
   - 미수금 현황
   - 3일/7일/이번달 입금 예정
6. 정산 현황
   - 미정산 건수
   - 파트너 정산 완료율
   - 임대인 정산 완료율
7. 세무 자료
   - 과세 매출 합계 (단기 Airbnb/Booking/Agoda)
   - 면세 매출 합계 (중기 ≥29일)
   - 부가세 예상액
   - 원천세 대상 (청소자 인건비 3.3%)
8. 다음 달 재무 전망
```

### CFO KPI (월간 달성 목표)

| KPI | 목표 | 측정 기준 |
|-----|------|---------|
| 순이익률 | ≥ 25% | (매출 - 총비용) / 매출 |
| 미정산 건수 | ≤ 5건 | settlement_missing status=open |
| 정산 완료율 | ≥ 95% | settled / total_settlements |
| 입금 예정 정확도 | ≥ 90% | actual_income / expected_deposit |
| 비용 초과 숙소 | ≤ 3개 | cost > revenue 숙소 수 |
| 세무 자료 제출 | 매월 10일 이전 | 전월 자료 마감일 |
| 미수금 회수율 | ≥ 90% | collected / expected |
| Founder 에스컬레이트 | ≤ 3건/월 | 정책 변경/고위험 미정산만 |

## 3. CTO 월간 연구 보고서

### 보고서 내용

```txt
1. 연구 요약 (한 달 핵심 기술 결정 3줄)
2. 기술 변경 이력
   - 주요 기능 추가/수정
   - 아키텍처 결정과 근거
   - 데이터 모델 변경
3. AI Level 현황
   - 7레벨 중 현재 위치
   - 이번 달 진척
4. 콘텐츠 파이프라인
   - 블로그 발행 수 (목표: 12건/월)
   - 에세이 발행 수 (목표: 4건/월)
   - 논문/백서 진행
5. 도시계획 가치 기록
   - MORO/HIERO 메시지 정합성
   - 연구소 콘텐츠 연결
6. 기술 부채/리스크
7. 다음 달 기술 로드맵
```

### CTO KPI (월간 달성 목표)

| KPI | 목표 | 측정 기준 |
|-----|------|---------|
| 블로그 발행 | ≥ 12건/월 | 주 3회 |
| 에세이 발행 | ≥ 4건/월 | 주 1회 |
| 문서화 완료 | ≥ 90% | 신규 기능 대비 문서 비율 |
| 기술 부채 해결 | ≥ 3건/월 | 해결된 기술 부채 |
| AI Level 진척 | 월 0.5+ | 7레벨 기준 진척도 |
| 메시지 정합성 | 100% | 브랜드 메시지 불일치 0건 |

## 4. 에스컬레이션 기준 (Execution → ETF → Founder)

### Execution → CEO 에스컬레이션

| 실행 역할 | 에스컬레이트 조건 | 이슈 유형 예시 |
|---------|--------------|-------------|
| 운영 | 예약 오류 반복, CS 미응답 24시간+, 체크인 당일 문제 | reservation_changed, cs_unanswered, checkin_problem |
| 운영 | 게스트 클레임 확대, 환불 요청 | guest_message (P0/P1) |
| 청소배정 | 청소 미배정 3시간+, 청소자 부족, 반복 지연 | cleaning_unassigned, cleaning_late |
| 현장 | 오픈 지연, 반복 민원, 시설 고장 긴급 | field_complaint, facility_broken (P0) |
| 마케팅 | 영업 병목, 우선순위 충돌, 리드 대량 유입 | new_lead (대량), priority_conflict |

### Execution → CFO 에스컬레이션

| 실행 역할 | 에스컬레이트 조건 | 이슈 유형 예시 |
|---------|--------------|-------------|
| 청소배정 | 추가비 정산, 청소비 이상 | cleaning_cost_missing |
| 현장 | 시설 수리비, 파트너 비용 | facility_broken (비용 연관) |
| 운영 | 환불 처리, 미수금 | transaction_review |

### Execution → CTO 에스컬레이션

| 실행 역할 | 에스컬레이트 조건 | 이슈 유형 예시 |
|---------|--------------|-------------|
| 마케팅 | 메시지 불일치, 브랜드 표현 문제 | content_needed, design_needed |
| 현장 | 사진 업데이트 필요 | photo_update_needed |

### ETF → Founder 에스컬레이션

| ETF 역할 | 에스컬레이트 조건 | 예시 |
|---------|--------------|------|
| CEO | 팀 간 우선순위 충돌, 반복 병목, 조직 구조 변경 필요 | cross_team_issue, execution_delay 반복 |
| CEO | 신규 계약/해지 최종 결정, 전략 방향 변경 | strategic_decision |
| CTO | MORO/HIERO 핵심 방향 확정, 브랜드 메시지 최종 결정 | core_direction_needed, moro_concept_needed |
| CTO | AI Level 단계 전환 결정 | strategic_decision |
| CFO | 세무 정책 기준 변경, 고위험 미정산 | tax_question (정책 변경), settlement_missing (critical) |
| CFO | 파트너 계약/신뢰 리스크 | partner_payment_issue (확대) |

### Founder 의사결정 기준

Founder는 다음 조건에 해당하는 건만 본다.

```txt
escalation_level = 'founder'
OR requires_final_decision = true
OR priority = 'P0'
```

최대 3건. 나머지는 ETF에서 처리.

## 5. KPI 평가 주기

```txt
일간: 이슈 해결률, 청소 완료율, 미처리 건수
주간: 팀원별 활동량, 에스컬레이션 횟수, 주간 해결 건수
월간: 전체 KPI 달성률, CEO/CTO/CFO 보고서, P&L
분기: AI Level 진척, 콘텐츠 파이프라인 달성, 사업 성장
```
