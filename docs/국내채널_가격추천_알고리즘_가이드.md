> 기준 구조: PriceLabs → Hostex → OTA 판매 흐름을 참고하되, 삼삼엠투는 별도 크롤링/분석/추천 구조로 설계한다.

## 0. 핵심 결론

PriceLabs는 Airbnb 등 글로벌 OTA의 시장 가격과 수요를 분석해 자동 가격 추천을 제공한다.  
하지만 **삼삼엠투(33m2)**, **리브애니웨이** 같은 국내 중단기 임대 채널은 PriceLabs/Hostex 자동 가격 체계로 완전히 커버되지 않는다.

따라서 HIERO는 별도의 국내 채널 가격 판단 엔진을 만들어야 한다.

```txt
삼삼엠투 크롤링 데이터
        ↓
주변 경쟁 숙소 분석
        ↓
실질 월 비용 계산
        ↓
우리 숙소의 예약 속도 / 공실률 결합
        ↓
가격 추천
        ↓
담당자 검토
        ↓
수동 반영
        ↓
변경 이력 축적
```

초기 목표는 **자동 가격 변경**이 아니다.

초기 목표는 다음이다.

```txt
1. 주변보다 비싼 숙소를 찾는다.
2. 예약이 안 잡히는 숙소를 찾는다.
3. 너무 싸게 팔리는 숙소를 찾는다.
4. 가격 변경 근거를 만든다.
5. 삼삼엠투 전용 가격 데이터베이스를 만든다.
```

## 2. HIERO의 역할

HIERO는 가격을 무조건 자동으로 바꾸는 시스템이 아니다.

HIERO의 역할은 다음이다.

| 구분 | 주체 | HIERO가 하는 것 |
|---|---|---|
| 가격 결정 | PriceLabs / HIERO | 추천가와 현재가 비교, 이상치 알림 |
| 채널 동기화 | Hostex | DB 캐시, 변경 이력 추적 |
| 판매 | Airbnb / 삼삼엠투 / OTA | 예약 데이터 수집, 매출 분석 |
| 수동 개입 | HIERO 운영팀 | 특정 날짜 / 특정 채널 가격 직접 변경 |
| 학습 | HIERO DB | 가격 변경 후 예약 결과를 축적 |

즉, HIERO는 **관찰자 + 의사결정 보조자**다.

## 3. 삼삼엠투 가격 분석이 필요한 이유

삼삼엠투는 Airbnb와 가격 구조가 다르다.

Airbnb는 주로 다음을 본다.

```txt
1박 가격
청소비
주말/평일 수요
단기 예약 속도
리뷰와 평점
```

삼삼엠투는 다음을 봐야 한다.

```txt
월 가격
일 가격
관리비
청소비
보증금
즉시입주 가능 여부
장기입주 할인
최소 계약 기간
생활권
옵션
예약 가능 기간
```

그래서 삼삼엠투의 가격 경쟁력은 단순 월세가 아니라 **실질 월 비용**으로 봐야 한다.

---

## 4. 삼삼엠투 실질 월 비용 계산

### 4-1. 기본 계산식

```txt
실질 월 비용 =
월 임대료
+ 관리비
+ 청소비 / 예상 체류 개월
+ 기타 필수 비용
- 장기 할인
- 즉시입주 할인
```

예시:

```txt
월 임대료: 1,200,000원
관리비: 100,000원
청소비: 50,000원
예상 체류 기간: 1개월
장기 할인: 100,000원

실질 월 비용 =
1,200,000 + 100,000 + 50,000 - 100,000
= 1,250,000원
```

### 4-2. 왜 실질 월 비용이 중요한가

고객은 월 가격만 보지 않는다.

삼삼엠투 고객은 실제로 다음처럼 판단한다.

```txt
"월세는 120만 원인데 관리비가 15만 원이네?"
"청소비까지 더하면 실제로 135만 원이네?"
"옆 숙소는 월세가 125만 원인데 관리비 포함이네?"
```

따라서 HIERO는 표면 가격이 아니라 고객이 체감하는 총비용을 분석해야 한다.

## 6. 경쟁 숙소 매칭 기준

삼삼엠투 가격 분석에서 가장 중요한 것은 **비교군을 잘 잡는 것**이다.

전체 삼삼엠투 숙소와 비교하면 의미가 없다.  
우리 숙소와 실제로 경쟁하는 숙소만 묶어야 한다.

---

### 6-1. 위치 기준

```txt
1차: 같은 동 / 같은 생활권
2차: 반경 500m
3차: 반경 1km
4차: 같은 지하철역 생활권
5차: 같은 구 단위
```

예시:

```txt
우리 숙소: 송파구 오금동

1차 경쟁군:
- 오금역
- 개롱역
- 방이역

2차 경쟁군:
- 송파
- 가락
- 문정
- 방이 생활권
```

---

### 6-2. 상품 유형 기준

```txt
원룸 → 원룸과 비교
투룸 → 투룸과 비교
쓰리룸 → 쓰리룸과 비교
오피스텔 → 오피스텔과 비교
빌라 → 빌라와 비교
아파트 → 아파트와 비교
```

---

### 6-3. 옵션 기준

삼삼엠투에서 중요한 옵션은 다음이다.

```txt
침대 수
퀸 침대 여부
TV 여부
책상 여부
세탁기
건조기
냉장고
주방 사용 가능 여부
엘리베이터
주차 가능 여부
반려동물 가능 여부
즉시입주 가능 여부
인터넷 / 와이파이
장기계약 가능 여부
```

옵션이 부족하면 가격을 낮추는 것이 아니라, 먼저 상품 개선이 필요할 수 있다.

예시:

```txt
가격은 주변보다 낮은데 예약이 안 잡힌다
→ 가격 문제가 아니라 사진/옵션/설명 문제일 수 있음
```

## 7. 분석 패키지 구조

PriceLabs에서 시장 가격을 분석하듯이, HIERO는 삼삼엠투용 분석 패키지를 만든다.

```txt
[1단계] 분석 패키지

숙소별 × 지역별 × 상품유형별 × 채널별
가격 / 공실 / 옵션 / ADR / 실질 월 비용 매트릭스
        ↓
"이 숙소는 삼삼엠투에서 주변 경쟁 숙소보다 12% 비싸다"
        ↓
가격 기준선 도출
```

분석 축은 다음과 같다.

| 축 | 데이터 소스 | 분석 내용 |
|---|---|---|
| 숙소별 | HIERO properties | 우리 숙소별 현재 가격 |
| 지역별 | 삼삼엠투 크롤링 | 생활권 평균가 / 중간가 |
| 상품유형별 | 삼삼엠투 크롤링 | 원룸 / 투룸 / 쓰리룸별 가격 |
| 옵션별 | 삼삼엠투 크롤링 | 옵션 유무에 따른 가격 차이 |
| 예약가능일 | 삼삼엠투 캘린더 | 경쟁 숙소 예약 속도 추정 |
| 내부성과 | reservations | 우리 숙소 가동률 / ADR |
| 비용구조 | property_costs | 최저가 하한선 계산 |

## 8. 가격 추천 알고리즘

### 8-1. 기본 기준가

기본 기준가는 경쟁 숙소의 실질 월 비용 중간값을 사용한다.

```txt
경쟁 숙소 실질 월 비용:
1,050,000
1,100,000
1,150,000
1,200,000
1,300,000
1,400,000

중간값: 1,175,000
하위 25%: 1,100,000
상위 25%: 1,300,000
```

신규 숙소 또는 예약이 부족한 숙소:

```txt
추천가 = 경쟁 중간값 × 0.95
```

성과가 좋은 숙소:

```txt
추천가 = 경쟁 중간값 × 1.05 ~ 1.15
```

---

### 8-2. 공실 기반 가격 조정

삼삼엠투는 중단기 체류 채널이므로 14일보다 **30일 공실률**이 중요하다.

```txt
향후 30일 공실률 기준

공실률 80% 이상 → EMERGENCY
공실률 60~80% → LOW
공실률 30~60% → NORMAL
공실률 30% 이하 → HIGH
```

| 등급 | 상태 | 가격 조정 |
|---|---|---|
| EMERGENCY | 거의 안 팔림 | 기준가 -15% ~ -25% |
| LOW | 예약 부족 | 기준가 -5% ~ -10% |
| NORMAL | 정상 | 기준가 유지 |
| HIGH | 예약 잘 됨 | 기준가 +5% ~ +15% |

---

### 8-3. 경쟁가 대비 보정

```txt
우리 숙소 실질 월 비용 > 경쟁 중간값 + 10%
→ 가격 인하 추천

우리 숙소 실질 월 비용 < 경쟁 중간값 - 10%
→ 가격 인상 검토

우리 숙소 실질 월 비용 ≈ 경쟁 중간값
→ 유지
```

예시:

```txt
경쟁 중간값: 1,200,000원
우리 숙소 실질 월 비용: 1,400,000원

차이: +16.7%

추천:
삼삼엠투 가격이 주변 경쟁군 대비 16.7% 높습니다.
예약 속도가 느리면 1,250,000~1,300,000원으로 조정 추천.
```

## 9. 최종 추천가 계산식

```txt
추천가 =
경쟁군 기준가
× 공실 보정
× 옵션 보정
× 위치 보정
× 내부성과 보정
× 시즌 보정
```

예시:

```txt
경쟁군 기준가: 1,200,000원
공실 보정: 0.90
옵션 보정: 1.05
위치 보정: 1.03
내부성과 보정: 0.95
시즌 보정: 1.00

추천가 =
1,200,000 × 0.90 × 1.05 × 1.03 × 0.95
= 약 1,055,000원
```

## 10. 점수화 구조

추천 가격을 바로 자동 적용하면 위험하다.  
먼저 점수화하고, 점수에 따라 액션을 다르게 한다.

```txt
삼삼엠투 가격 추천 점수 = 100점 만점
```

| 항목 | 배점 | 설명 |
|---|---:|---|
| 가격 경쟁력 | 30점 | 주변 대비 비싼지 싼지 |
| 예약 가능성 | 25점 | 향후 30일 공실률 |
| 옵션 경쟁력 | 15점 | 침대, TV, 세탁기, 주방 등 |
| 위치 경쟁력 | 15점 | 역세권, 생활권, 주변 수요 |
| 사진/리뷰 품질 | 10점 | 클릭/전환 가능성 |
| 시즌/이벤트 | 5점 | 명절, 행사, 입학, 출장 시즌 |

점수별 액션:

| 점수 | 상태 | 액션 |
|---:|---|---|
| 85점 이상 | 가격 인상 가능 | +5~15% 검토 |
| 70~84점 | 적정 | 유지 |
| 50~69점 | 개선 필요 | -5~10% 또는 옵션/사진 개선 |
| 50점 미만 | 위험 | 가격 인하 + 상품 개선 필요 |

## 11. 가격 판단 로직

### 11-1. 가격이 비싼데 예약도 안 잡히는 경우

```txt
조건:
- 경쟁군 대비 +10% 이상 비쌈
- 향후 30일 공실률 60% 이상

판단:
가격 인하 필요

액션:
- 추천가 제안
- 담당자 알림
- 삼삼엠투 가격 수정 검토
```

---

### 11-2. 가격이 싼데 예약이 잘 잡히는 경우

```txt
조건:
- 경쟁군 대비 -10% 이상 저렴
- 향후 30일 공실률 30% 이하

판단:
가격 인상 가능
```

## 12. 삼삼엠투 특화 판단 요소

### 12-1. 즉시입주 할인

삼삼엠투는 즉시 입주 가능한 방의 전환율이 중요하다.

```txt
공실 시작일이 7일 이내
+ 예약 없음
+ 경쟁군 많음
→ 즉시입주 할인 제안
```

예시:

```txt
현재 월 가격: 1,300,000원
추천:
즉시입주 할인 적용
1개월 첫 달 1,190,000원 제안
```

---

### 12-2. 장기입주 할인

삼삼엠투는 장기 체류 수요가 강하므로 기간별 가격 전략이 필요하다.

```txt
1개월 가격
2개월 가격
3개월 이상 가격
```

예시:

```txt
1개월: 1,300,000원
2개월: 월 1,250,000원
3개월 이상: 월 1,200,000원
```

추천 로직:

```txt
공실률 높음 + 장기 수요 지역
→ 2개월 이상 할인 제안

예약 잘 잡힘 + 공급 부족 지역
→ 장기할인 축소
```

---

### 12-3. 관리비 포함/별도 표기

고객은 관리비 별도 가격에 민감하다.

```txt
A숙소: 월 120만 원 + 관리비 15만 원
B숙소: 월 128만 원 관리비 포함

표면 가격은 A가 싸지만,
실질 가격은 B가 유리할 수 있음
```

따라서 HIERO는 관리비 포함 여부를 반드시 정규화한다.

---

### 12-4. 보증금 영향

보증금은 월 비용에는 직접 반영하지 않더라도 전환율에 영향을 준다.

```txt
보증금 높음
→ 예약 전환율 하락 가능

보증금 낮음
→ 진입 장벽 낮음
```

초기 점수화에서는 보증금을 별도 감점 요소로 둔다.

## 17. 자동화 단계

### Phase 1. 분석만 한다

```txt
삼삼엠투 크롤링
→ 경쟁 숙소 분석
→ 실질 월 비용 계산
→ 우리 숙소와 비교
→ 대시보드 표시
```

이 단계에서는 가격을 바꾸지 않는다.

---

### Phase 2. 알림을 보낸다

```txt
예약 안 잡히는 숙소
경쟁 대비 비싼 숙소
공실 위험 숙소
가격 인상 가능 숙소
```

이런 숙소만 알림으로 올린다.

---

### Phase 3. 사람이 검토 후 수동 반영

```txt
HIERO 추천가 확인
→ 담당자 삼삼엠투 접속
→ 가격 수정
→ 변경 이력 입력
```

---

### Phase 4. 반자동화

향후 가능해지면 다음 단계로 간다.

```txt
추천가 생성
→ 담당자 승인
→ 삼삼엠투 가격 자동 수정 또는 반자동 입력
```

## 18. 핵심 안전장치

가격 자동화에서 가장 중요한 것은 **절대 손실 구간으로 내려가지 않는 것**이다.

```txt
1. 숙소별 최저가 하한선 설정
2. 손익분기점 이하 가격 금지
3. 자동 변경 전 담당자 승인
4. 모든 변경 로그 기록
5. 하루 가격 변경 횟수 제한
6. 급격한 인하 제한
7. 수동 오버라이드 가능
8. 추천가와 실제 반영가 구분
9. 가격 변경 후 예약 결과 추적
```

예시:

```txt
숙소: 오금 205호
월 고정비: 950,000원
최소 목표 마진: 200,000원
최저가 하한선: 1,150,000원

추천가가 1,050,000원이어도
시스템은 1,150,000원 아래로 추천하지 않는다.
```

## 24. 운영 매뉴얼 문구

```txt
삼삼엠투 가격 분석 엔진은 PriceLabs가 커버하지 못하는 국내 중단기 임대 채널의 가격 의사결정을 보완하기 위한 HIERO 내부 가격 추천 시스템이다.

삼삼엠투의 주변 경쟁 숙소 가격, 관리비, 청소비, 할인, 옵션, 예약 가능 상태를 수집하고, 이를 HIERO 내부 예약 데이터와 결합하여 숙소별 적정 가격을 추천한다.

초기 단계에서는 자동 가격 변경이 아니라 추천과 알림만 제공하며, 담당자가 검토 후 수동으로 가격을 반영한다.

핵심 목표는 다음과 같다.

1. 공실 위험 숙소를 조기에 발견한다.
2. 주변 경쟁 숙소 대비 비싼 숙소를 감지한다.
3. 예약이 빠르게 차는 숙소는 가격 인상 가능성을 발견한다.
4. 가격 변경 이력을 축적하여 HIERO만의 국내 채널 가격 데이터베이스를 만든다.
5. 장기적으로 PriceLabs가 하지 못하는 국내 채널 전용 가격 추천 엔진을 구축한다.
```

## 25. 한 줄 요약

```txt

주변 경쟁 숙소의 실질 월 비용

+ 향후 공실 위험
+ 옵션 경쟁력
+ 손익분기점 하한선
=
HIERO 추천 가격
```
