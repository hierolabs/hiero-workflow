# 개인 업무 목표 + 자동 달성 추적 시스템

> 작성: 2026년 5월 8일
> 목적: 각 팀원이 자기 주도적으로 업무 목표를 세우고, 시스템이 자동으로 달성률을 채워주는 구조

---

## 1. 핵심 개념

```txt
목표 설정 = 수동 (본인이 세움)
달성 추적 = 자동 (시스템이 채움)
동기부여 = 시각화 (트리 + 진행률 + 팀 비교)
```

## 3. 자동 달성 측정 기준

### 측정 가능한 항목 (시스템에서 자동 카운트)

| 목표 카테고리 | 자동 측정 소스 | 쿼리 기준 |
|-------------|-------------|---------|
| 이슈 해결 | issues.status=resolved, assignee_name=나 | 기간 내 resolved_at |
| 리드 연락 | leads.status 변경 이력 | 기간 내 activity_log |
| 제안서 발송 | leads.status=proposal_sent | 기간 내 |
| OTA 등록 | property_platforms.status 변경 | 기간 내 |
| 소개글 작성 | onboarding_checks (Phase 3 체크) | 기간 내 checked_at |
| 사진 촬영 | onboarding_checks (Phase 2 체크) | 기간 내 checked_at |
| 청소 배정 | cleaning_tasks.status=assigned, 배정자=나 | 기간 내 |
| 청소 완료 확인 | cleaning_tasks.status=completed | 기간 내 |
| CS 응대 | conversations 메시지 발송 수 | 기간 내 |
| 정산 확인 | activity_log action=settlement 관련 | 기간 내 |
| 세팅 완료 | onboarding_checks (Phase 1 체크) | 기간 내 |
| 에스컬레이트 처리 | issues.escalated_from=나, resolved | 기간 내 |

### 수동 입력 항목

| 목표 | 수동 이유 |
|------|---------|
| 미팅 진행 | 외부 활동, 시스템 외 |
| 전화 상담 | 통화 기록 미연동 시 |
| 현장 방문 | GPS 미연동 시 |

## 4. 데이터 모델

### weekly_goals 테이블

```sql
weekly_goals (
  id,
  user_id,                -- 목표 소유자
  week_start,             -- 주 시작일 (월요일)
  week_end,               -- 주 종료일 (일요일)
  category,               -- supply, digital, operations, finance, custom
  title,                  -- "OTA 등록 3건"
  target_count,           -- 목표 수 (3)
  current_count,          -- 현재 달성 수 (자동 업데이트)
  measure_type,           -- auto, manual
  measure_source,         -- issues_resolved, leads_contacted, onboarding_phase3, cleaning_assigned...
  measure_query,          -- JSON: {"status":"resolved","assignee":"나"}
  is_completed,           -- 100% 달성 여부
  sort_order,             -- 표시 순서
  created_at,
  updated_at
)
```

### goal_templates 테이블 (역할별 기본 목표 템플릿)

```sql
goal_templates (
  id,
  role_title,             -- operations, cleaning_dispatch, marketing, field, ceo, cto, cfo
  category,
  title,
  default_target,         -- 기본 목표 수
  measure_type,
  measure_source,
  sort_order
)
```

## 5. 역할별 기본 목표 템플릿

### 운영 (재관)
```txt
├── 이슈 해결 10건/주          auto: issues_resolved
├── CS 응대 20건/주            auto: messages_sent
├── 체크인 문제 0건/주          auto: issues_type=checkin (역방향)
└── 미처리 이슈 3건 이하       auto: issues_open (역방향)
```

### 청소배정 (우현)
```txt
├── 청소 배정 완료율 95%/주    auto: cleaning_assigned / cleaning_total
├── 사진 확인 완료율 90%       auto: photo_checked
├── 배정 메시지 발송 100%      auto: cleaning_dispatched
└── 미배정 0건 유지            auto: cleaning_pending (역방향)
```

### 마케팅 (예린)
```txt
├── 리드 연락 10건/주          auto: leads_contacted
├── 제안서 발송 3건/주         auto: leads_proposal
├── OTA 등록 2건/주           auto: platforms_registered
├── 소개글 작성 3건/주         auto: onboarding_phase3
└── 콘텐츠 제작 2건/주         manual
```

### 현장 (진태)
```txt
├── 세팅 완료 2건/주           auto: onboarding_phase1
├── 시설 이슈 해결 5건/주      auto: issues_facility_resolved
├── 사진 촬영 2건/주           auto: onboarding_phase2
└── 민원 처리 24h 이내         auto: avg_resolve_time
```

### CEO (지훈)
```txt
├── 팀 병목 해소 3건/주        auto: issues_bottleneck_resolved
├── 에스컬레이트 처리 5건/주   auto: issues_escalated_resolved
├── 지연 업무 0건 유지         auto: issues_delayed (역방향)
└── 리드 전환 2건/주           auto: leads_converted
```

### CFO (수빈)
```txt
├── 정산 완료율 95%            auto: settlement_completed
├── 미정산 5건 이하            auto: settlement_pending (역방향)
├── 세무 자료 제출             manual
└── 비용 승인 처리 3건/주      auto: issues_cost_approved
```

## 6. 에스컬레이션 → 목표 자동 반영 흐름

```txt
재관이 이슈 해결함
  → issues.status = resolved
  → ActivityLog 자동 생성
  → weekly_goals (measure_source=issues_resolved) current_count++
  → 마이페이지에 즉시 반영

예린이 OTA 등록함
  → onboarding_checks Phase 4 체크
  → ActivityLog 자동 생성
  → weekly_goals (measure_source=onboarding_phase4) current_count++
  → 마이페이지에 즉시 반영

진태가 시설 이슈 해결함
  → issues.status = resolved, type=facility
  → ETF에서 에스컬레이트 처리 완료
  → weekly_goals (measure_source=issues_facility_resolved) current_count++
```

## 7. 동기부여 요소

### 완료 시 시각적 보상
```txt
목표 100% 달성 → ✓ 체크 + 초록색 전환 + 축하 애니메이션
전체 목표 달성 → ★ 주간 MVP 배지
팀 1위 → 🏆 표시
```

### 팀 비교 (투명성)
```txt
전체 팀원 달성률 비교 → 건전한 경쟁
단, 순위가 아닌 "달성률" 중심 → 각자 목표가 다르므로
```

### Founder/CEO 뷰
```txt
팀 전체 주간 목표 달성률 → CEO Board에 표시
미달성 항목 → 병목 파악 → 지원/조정
개인별 추이 → 월간 평가 근거
```

## 8. 개발 순서

```txt
Phase 1. DB + API (1주)
  - weekly_goals, goal_templates 테이블
  - 목표 CRUD API
  - 자동 달성 카운트 서비스 (매 시간 또는 실시간)

Phase 2. 마이페이지 UI (3일)
  - 트리형 목표 뷰 + 진행률 바
  - 간트차트 (일별 달성)
  - 팀 비교 달성률

Phase 3. 역할별 템플릿 (2일)
  - 기본 목표 자동 생성 (매주 월요일)
  - 본인이 수정/추가 가능

Phase 4. Founder/CEO 연결 (2일)
  - CEO Board에 팀 목표 달성률
  - Founder Brief에 미달성 알림
```
