# ETF 월간 보고서 자동화 개발 지시서

> 작성: 2026년 5월 8일
> 목적: ETF(CEO/CTO/CFO)가 월 1회 자동 생성하는 경영/연구/재무 보고서 시스템 구축
> AI Level: Level 7 (Auto Generation)

---

## 1. 개요

HIERO의 운영 데이터(Level 2~4)를 기반으로 ETF 3명이 각각의 보고서를 월 1회 자동 생성한다.

```txt
CEO → 전체 운영/경영 보고서
CTO → 연구 보고서
CFO → 재무 보고서 + 세무 신고 자료
```

보고서는 AI 파이프라인(GPT→Claude→Gemini)으로 초안을 생성하고, 사람이 검토/수정 후 확정한다.

---

## 2. AI 기술 활용 7레벨 체계

```txt
Level 1. 수동 입력 (Manual)
Level 2. 자동 수집 (Auto Capture)           ✅ 완료
Level 3. 자동 분류 (Auto Classification)     ✅ 완료
Level 4. 자동 판단 (Auto Decision Support)   ⚡ 부분 구현
Level 5. 자동 실행 (Auto Execution)          — 다음
Level 6. 자동 학습 (Auto Learning)           — 향후
Level 7. 자동 생성 (Auto Generation)         ← 이 지시서
```

Level 7은 Level 5~6 없이도 구현 가능하다. 이유:

```txt
보고서 = 기존 데이터 집계 + AI 텍스트 생성
실행 자동화(Level 5) = 의사결정 + 실행 + 피드백 루프 필요
→ 보고서가 더 단순하고 안전
```

## 3. CEO 경영 보고서

### 데이터 소스

```txt
issues 테이블 → 이슈 통계 (유형별, 우선순위별, 팀원별)
reservations 테이블 → 예약 통계 (채널별, 숙소별, 가동률)
hostex_transactions 테이블 → 매출/정산 요약
activity_logs 테이블 → 팀원별 활동량, 에스컬레이션 횟수
property_business_diagnosis 테이블 → 5엔진 진단 점수
notifications 테이블 → 의사결정 이력
```

### 보고서 구조

```txt
1. 경영 요약 (Executive Summary)
   - 한 달 핵심 3줄

2. KPI 대시보드
   - 가동률, ADR, RevPAR, 매출, 비용, 순이익
   - 전월 대비 변화율

3. 팀 성과
   - 팀원별 이슈 처리 건수, 해결 시간, 에스컬레이션
   - 병목 구간 분석

4. 운영 이슈 분석
   - 반복 이슈 Top 5
   - 신규 이슈 유형
   - 해결률, 미해결 누적

5. 숙소 포트폴리오
   - 5엔진 진단 요약
   - 문제 숙소 리스트
   - 계약 만료 예정

6. 의사결정 기록
   - Founder Brief에 올라간 건
   - 승인/반려/돌려보내기 이력

7. 다음 달 액션 플랜
   - AI 추천 우선순위
```

## 4. CTO 연구 보고서

### 데이터 소스

```txt
activity_logs 테이블 → 기술 관련 활동 (documentation, research, message)
issues 테이블 → CTO 배정 이슈 + 해결 기록
git log → 코드 변경 이력 (커밋 메시지, 파일 변경 수)
CLAUDE.md / DATA_ARCHITECTURE.md → 기술 결정 기록
콘텐츠 파이프라인 현황 → 블로그/에세이/논문 발행 현황
```

### 보고서 구조

```txt
1. 연구 요약
   - 한 달 핵심 기술 결정 3줄

2. 기술 변경 이력
   - 주요 기능 추가/수정
   - 아키텍처 결정과 근거

3. 데이터 아키텍처 현황
   - Data 1/2/3 구조 변경 사항
   - 새로운 테이블/필드

4. 콘텐츠 파이프라인 현황
   - 블로그 발행 수 (목표: 주3회 = 월12건)
   - 에세이 발행 수 (목표: 주1회 = 월4건)
   - 논문/백서 진행 현황

5. 도시계획 가치 기록
   - MORO/HIERO 메시지 정합성
   - 연구소 콘텐츠 연결 현황

6. 기술 부채 및 리스크
   - 미구현 기능
   - 성능 이슈
   - 보안 점검

7. 다음 달 기술 로드맵
```

## 5. CFO 재무 보고서 + 세무 신고 자료

### 데이터 소스

```txt
Data 1 (reservations) → reservation_date 기준 매출
Data 2 (hostex_transactions + cost_raw + cost_allocations) → 비용 전체
Data 3 (JOIN) → deposit_date 기준 입금 예정
settlement 집계 → 숙소별 P&L
세무 분류 → 과세/면세 구분 (단기/중기/서비스)
```

### 재무 보고서 구조

```txt
1. 재무 요약
   - 총매출, 총비용, 순이익, 전월 대비

2. 매출 분석
   - 예약일 기준 매출 (reservation_date)
   - 입금 예정 (deposit_date)
   - 실제 입금 (transaction_at)
   - 채널별 매출 비중

3. 비용 분석
   - cost_allocations 기준 월별 배분
   - 청소비, 월세, 관리비, 수리비 항목별
   - 비용 증감 분석

4. 숙소별 P&L
   - 수익률 TOP/BOTTOM 5
   - 적자 숙소 분석
   - 권역별 수익률

5. 현금 흐름
   - 입금 예정 vs 실제 입금 차이
   - 미수금 현황
   - 다음 달 입금 예정

6. 세무 자료
   - 과세/면세 분류 집계
   - 단기임대 과세 (Airbnb, Booking, Agoda)
   - 중기임대 과세 (<29일) / 면세 (≥29일)
   - 부가세 예상액
```

### 세무 신고 자료 구조

```txt
1. 부가가치세 신고 자료
   - 과세 매출 합계
   - 면세 매출 합계
   - 매입세액 (비용 중 세금계산서 발행분)

2. 원천세 자료
   - 청소자 인건비
   - 3.3% 원천징수 대상

3. 계정과목 분류
   - 면세 전대임대료 (4101)
   - 과세 숙박매출 (4201)
   - 과세 서비스매출 (4301~)
   - 직접 운영비 (5101~5114)
   - 일반관리비 (6101~6117)
```

## 6. 자동화 프로세스

### 생성 흐름

```txt
매월 1일 00:00 (자동 트리거 또는 수동 실행)
    ↓
1. 데이터 집계 (Backend)
   - 전월 1일 ~ 말일 기준
   - issues, reservations, transactions, activity_logs, cost_allocations 집계
   - JSON 형태로 정리
    ↓
2. AI 초안 생성 (AI Pipeline)
   - GPT-4o: 보고서 구조화 + 초안 작성
   - Claude: 데이터 검증 + 인사이트 추가
   - Gemini: 팩트체크 + 최종 검수
    ↓
3. 초안 저장
   - DB에 report_drafts 테이블로 저장
   - 상태: draft → reviewed → confirmed
    ↓
4. ETF 검토
   - CEO/CTO/CFO 각자 보드에서 초안 확인
   - 수정/승인
    ↓
5. 확정 및 배포
   - PDF 생성
   - Founder에게 전달
   - 서울 AI 도시계획 연구소 콘텐츠로 변환 (CTO 보고서)
```

### 필요 테이블

```sql
report_drafts (
  id,
  report_type,     -- ceo_monthly, cto_monthly, cfo_monthly, cfo_tax
  period,          -- "2026-04"
  role_title,      -- ceo, cto, cfo
  status,          -- draft, reviewed, confirmed
  data_json,       -- 집계 데이터 원본
  content,         -- AI 생성 보고서 본문 (Markdown)
  reviewed_by,
  reviewed_at,
  confirmed_at,
  created_at,
  updated_at
)
```

### 필요 API

```txt
POST /admin/reports/generate        — 월간 보고서 생성 (트리거)
GET  /admin/reports                  — 보고서 목록
GET  /admin/reports/:id              — 보고서 상세
PATCH /admin/reports/:id/review      — 검토 완료
PATCH /admin/reports/:id/confirm     — 최종 확정
GET  /admin/reports/:id/export       — PDF 다운로드
```

### 필요 프론트엔드

```txt
/admin/src/pages/etf/Reports.tsx     — 보고서 목록 + 상세 뷰
각 ETF Board에 "이번 달 보고서" 섹션 추가
```


## 8. 콘텐츠 파이프라인 연결

보고서는 HIERO 내부용으로 끝나지 않는다.

```txt
CEO 경영보고서 → 투자자 업데이트, 내부 공유
CTO 연구보고서 → 서울 AI 도시계획 연구소 월간 리서치 노트
               → 블로그/에세이 소재
               → 분기 강의자료 원천
               → 연간 백서/책 원고
CFO 재무보고서 → 세무사 전달 자료
               → 임대인 정산 리포트
               → 투자자 재무 업데이트
```
