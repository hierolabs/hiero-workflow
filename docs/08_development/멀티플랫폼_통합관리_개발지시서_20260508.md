# 멀티플랫폼 통합 관리 + 대화 자동 분석 개발 지시서

> 작성: 2026년 5월 8일
> 목적: 삼삼엠투/리브애니웨어/자리톡 등 API 미지원 국내 플랫폼을 HIERO에서 통합 관리하고, 고객 대화를 자동 분석하여 이슈를 추출한다.
> 담당: 운영(오재관) 70%, 청소배정(김우현) 20%, 현장(김진태) 10%

---

## 1. 현재 문제

```txt
재관의 하루:

06:00 삼삼엠투 호스트 페이지 로그인 → 신규 예약 확인
06:10 리브애니웨어 콘솔 로그인 → 신규 예약 확인
06:20 자리톡 로그인 → 신규 예약 확인
06:30 Hostex 열기 → 3개 플랫폼 예약을 수동 입력
07:00 각 플랫폼 메시지함 확인 → 게스트 문의 응대
      삼삼엠투 메시지함에서 응대
      리브 메시지함에서 응대
      자리톡 메시지함에서 응대
      Airbnb는 Hostex에서 자동
...반복

문제:
1. 매일 4~5개 창을 열고 각각 로그인
2. 예약을 Hostex에 수동 복사 (누락 위험)
3. 메시지 응대를 플랫폼별로 따로 (일관성 없음)
4. 고객 대화에서 이슈(가격/주차/시설)가 발생해도 추적 안 됨
5. 같은 질문을 반복 응대 (FAQ 자동화 없음)
```

---

## 2. 목표

```txt
1. HIERO 하나로 모든 플랫폼 예약을 관리한다
2. 고객 대화를 자동 분석하여 이슈를 추출한다
3. 반복 질문은 AI가 답변 초안을 제안한다
4. 추출된 이슈는 피라미드 에스컬레이션과 연결된다
```

## 3. Phase 1: 플랫폼 허브 + 수동 예약 직접 입력

### 3-1. 수동 예약 등록 API

현재 예약은 Hostex API로만 들어온다. 국내 플랫폼 예약을 HIERO에 직접 등록하는 API를 추가한다.

```txt
POST /admin/reservations/manual
```

Request:

```json
{
  "property_id": 42,
  "guest_name": "홍길동",
  "guest_phone": "010-1234-5678",
  "check_in_date": "2026-05-15",
  "check_out_date": "2026-06-15",
  "total_rate": 1500000,
  "channel": "삼삼엠투",
  "source": "manual",
  "memo": "삼삼엠투 예약번호 SM-20260510-001"
}
```

로직:

```txt
1. reservations 테이블에 저장
2. status = "accepted"
3. channel_name = "삼삼엠투" | "리브애니웨어" | "자리톡" | "직접계약" | "기타"
4. source = "manual" (Hostex API와 구분)
5. 청소 태스크 자동 생성 (체크아웃일 기준)
6. 캘린더에 즉시 반영
7. ActivityLog 기록
8. 알림: 청소배정(김우현)에게 "새 예약 등록" 알림
```

## 4. Phase 2: 고객 대화 자동 분석 시스템

### 4-1. 대화 이슈 자동 추출

현재 게스트 메시지는 Hostex에서 동기화되어 conversations/messages 테이블에 저장된다.
이미 message_analysis_service.go에 키워드 기반 이슈 감지가 있다.

이를 확장하여 **모든 플랫폼 대화를 분석**한다.

### 4-2. 이슈 카테고리

```txt
가격 이슈 (price)
  키워드: 할인, 가격, 비싸, 저렴, 장기할인, 네고, 깎아
  액션: CFO 알림 + 가격 조정 판단 요청
  에스컬레이션: 운영 → CFO

주차 이슈 (parking)
  키워드: 주차, 차, 주차장, 주차비, 차량
  액션: 숙소별 주차 정보 자동 안내
  에스컬레이션: 운영 → 현장 (김진태)

시설 이슈 (facility)
  키워드: 고장, 안됨, 수리, 에어컨, 보일러, 온수, 세탁기, 와이파이
  액션: 이슈 자동 생성 + 현장 담당자 배정
  에스컬레이션: 운영 → 현장 (김진태) → CEO/CFO

청결 이슈 (cleaning)
  키워드: 더럽, 청소, 먼지, 곰팡이, 냄새, 머리카락, 벌레
  액션: 이슈 자동 생성 + 청소배정 담당자 알림
  에스컬레이션: 운영 → 청소배정 (김우현)

체크인 이슈 (checkin)
  키워드: 비밀번호, 도어락, 문, 열쇠, 못 들어, 위치, 찾기 어려
  액션: 체크인 정보 자동 재전송
  에스컬레이션: 운영 → 현장 (김진태)

퇴실 이슈 (checkout)
  키워드: 퇴실, 체크아웃, 연장, 레이트, 짐 보관
  액션: 연장 가능 여부 확인 + 가격 안내
  에스컬레이션: 운영 → CEO (일정 충돌 시)

환불 이슈 (refund)
  키워드: 환불, 취소, 계약해지, 위약금
  액션: 이슈 자동 생성 (P1)
  에스컬레이션: 운영 → CFO

칭찬/감사 (positive)
  키워드: 감사, 좋았, 추천, 만족, 깨끗, 친절
  액션: 리뷰 요청 메시지 자동 제안
  에스컬레이션: 없음 (기록만)

재계약 의향 (renewal)
  키워드: 연장, 다음 달, 재계약, 계속 살고
  액션: 재계약 리드 자동 생성 + CEO 알림
  에스컬레이션: 운영 → CEO
```

### 4-3. 분석 프로세스

```txt
게스트 메시지 수신 (Hostex 웹훅 or 수동 입력)
    ↓
AI 대화 분석
    ↓
이슈 카테고리 분류 (price/parking/facility/cleaning/checkin/checkout/refund/positive/renewal)
    ↓
심각도 판정 (info / warning / critical)
    ↓
자동 액션
  ├── info: 기록만 (ActivityLog)
  ├── warning: 이슈 생성 + 담당자 알림
  └── critical: 이슈 생성 (P0/P1) + 에스컬레이션
    ↓
AI 답변 초안 제안
  ├── FAQ 매칭: 기존 답변 템플릿 제안
  └── 신규 질문: AI가 초안 생성 → 재관이 수정 후 전송
    ↓
대화 로그에 분석 결과 태깅
    ↓
반복 패턴 누적 → 주간/월간 보고서 반영
```

### 4-4. 데이터 모델

```sql
-- 대화 분석 결과
conversation_analyses (
  id,
  conversation_id,     -- conversations 테이블 FK
  message_id,          -- 트리거된 메시지
  category,            -- price, parking, facility, cleaning, checkin, checkout, refund, positive, renewal
  severity,            -- info, warning, critical
  summary,             -- AI 요약 (한 줄)
  keywords_matched,    -- 매칭된 키워드들
  auto_action,         -- 자동 수행된 액션
  issue_id,            -- 생성된 이슈 (있으면)
  suggested_reply,     -- AI 제안 답변
  property_id,
  created_at
)

-- FAQ 답변 템플릿
reply_templates (
  id,
  category,            -- price, parking, facility 등
  trigger_keywords,    -- 트리거 키워드 (JSON array)
  template_text,       -- 답변 템플릿
  property_specific,   -- 숙소별 커스텀 여부
  property_id,         -- NULL이면 공통
  use_count,           -- 사용 횟수
  created_at,
  updated_at
)
```

### 4-5. API

```txt
-- 대화 분석
POST /admin/conversations/:id/analyze     — 특정 대화 AI 분석 실행
GET  /admin/conversations/:id/analysis    — 분석 결과 조회
GET  /admin/conversation-analyses         — 전체 분석 목록 (필터: category, severity, date)
GET  /admin/conversation-analyses/stats   — 이슈 카테고리별 통계

-- FAQ 템플릿
GET  /admin/reply-templates               — 템플릿 목록
POST /admin/reply-templates               — 템플릿 추가
PUT  /admin/reply-templates/:id           — 템플릿 수정
GET  /admin/reply-templates/suggest?category=parking&property_id=42  — 답변 제안
```

## 5. Phase 3: AI 답변 자동 제안

### 5-1. 답변 제안 흐름

```txt
게스트: "주차장 있나요?"
    ↓
AI 분석: category=parking, severity=info
    ↓
reply_templates에서 매칭:
  → 숙소별 주차 정보 + 템플릿 조합
    ↓
제안 답변:
  "안녕하세요! [숙소명]은 건물 내 주차장이 있습니다.
   지하 1층 [호수] 전용 주차 1대 무료입니다.
   추가 차량은 인근 [공영주차장명]을 이용하시면 됩니다."
    ↓
재관이 확인 → [전송] 또는 [수정 후 전송]
```

### 5-2. 숙소별 정보 자동 바인딩

답변 템플릿에 숙소별 변수를 자동 삽입:

```txt
{property_name} — 숙소명
{address} — 주소
{parking_info} — 주차 정보
{wifi_password} — 와이파이 비번
{checkin_time} — 체크인 시간
{checkout_time} — 체크아웃 시간
{door_password} — 도어락 비밀번호
{trash_info} — 쓰레기 배출 안내
{nearby_facilities} — 주변 시설
```

이 변수들은 properties 테이블에 저장하고, 답변 생성 시 자동 치환한다.

## 6. 기존 시스템 연결 포인트

### 피라미드 에스컬레이션 연결

```txt
대화에서 추출된 이슈 → Issue 자동 생성
  → 이슈 유형별 자동 배정 (AssignIssueByType)
  → 알림 + ActivityLog 자동 생성
  → 심각도 critical → 에스컬레이트
```

### ETF 월간 보고서 연결

```txt
conversation_analyses 통계 → CEO 경영 보고서
  "이번 달 가격 이슈 23건, 시설 이슈 15건, 주차 문의 42건"
  "가장 많은 이슈: 주차(42건) → 주차 안내 자동화 도입 후 문의 60% 감소"
```

### CTO 콘텐츠 파이프라인 연결

```txt
대화 분석 패턴 → CTO 연구 보고서
  "고객이 가장 많이 묻는 질문 Top 10"
  "채널별 문의 유형 차이 분석"
  → 블로그/에세이 소재
```

### CFO 재무 보고서 연결

```txt
가격 이슈 분석 → CFO 재무 보고서
  "할인 요청 건수: 23건, 평균 할인율: 8.5%"
  "환불 요청: 3건, 환불 금액: 450,000원"
```

### AI Level 연결

```txt
Phase 1 (플랫폼 허브 + 수동 입력) = Level 2 확장
Phase 2 (대화 자동 분석) = Level 3 확장
Phase 3 (답변 자동 제안) = Level 4 확장
향후: Puppeteer 자동 동기화 = Level 5
향후: 대화 패턴 학습 = Level 6
향후: 보고서 자동 생성 = Level 7
```
