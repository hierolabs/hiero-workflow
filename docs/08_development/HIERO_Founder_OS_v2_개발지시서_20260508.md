## 0. 핵심 원칙

### 김진우 손에서 빠져야 할 6가지 (31,867건 분석 결과)
1. 확인했는지 확인 → **자동 완료 체크**
2. 담당자가 누구인지 → **자동 배정**
3. 처리됐는지 확인 → **State Transition 추적**
4. 사진/리스트 공유 → **업로드 의무화 + 미제출 알림**
5. 정산/입금/환불 → **정산 자동 확인**
6. 고객 응대 방향 → **AI 감지 + 대응 제안**

### KPI
> **김진우가 다시 말하지 않아도 처리되는 비율**

```
Messages 페이지에 표시
  ↓
[AI 대응] → CS Agent가 응답 초안 생성 (CS 지식베이스 참조)
[이슈 등록] → Issue 자동 생성 + 자동 배정
[무시] → dismissed
  ↓
대응 시작 → responding (응답 시각 기록)
  ↓
대응 완료 → resolved (해결 시각 + 유형 기록)
```

### 6대 감지 카테고리
| 카테고리 | 키워드 예시 | 긴급도 |
|---------|-----------|--------|
| checkin | 비밀번호, 출입, 체크인, 얼리 | medium |
| parking | 주차, 출차, 차량 | medium |
| boiler | 온수, 보일러, 난방, 에러코드 | high |
| cleaning | 청소, 더러, 냄새, TV, 와이파이 | medium |
| reservation | 연장, 환불, 취소, 결제 | low |
| emergency | 위험, 화재, 침입, 119 | critical |

### CS 지식베이스 (DB 저장 완료)
- `manual_entries` 테이블: cs/checkin, cs/parking, cs/boiler, cs/cleaning, cs/reservation, cs/emergency
- 각 항목: 반복 이슈 → 1차 대응 → 에스컬레이션 조건 → 예방 조치
