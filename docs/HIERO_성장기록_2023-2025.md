## 1단계: 자체 운영 (2023.11 ~ 2024.5)

### 사람
- 김진우 (나): 예약, 전략, 판단
- 김진태 (동생): 셋팅, 청소, 플랫폼 등록 (판단 없이 실행)
- 박수빈 (와이프): 초기 청소, 이후 정산/데이터
- 3명이 매일 청소 3~5건 직접 수행

### 타임라인
- **23.11.24**: 첫 예약 — 삼삼엠투, 1주일, ₩24만
- **23.12**: 5개 한번에 공급. 와이프와 이사. 학원 개강. 김지훈 합류
- **23년 매출**: ₩500만
- **24.1~4**: 월 2채씩 증가. 삼투만으로 성장
- **24.4**: 와이프와 깊은 대화 → "매월 모든 요소 10%씩 증가" 전략 확정
- **24.5**: 비엔비 시작. 무허가 퇴출 선언. 월 5채씩 공격적 확대

### 핵심 발견 1: 인벤토리 풀 (방 변경 배정)
예건네스빌 702호, 802호를 동시에 여러 플랫폼에 등록했다.
같은 숙소를 2명이 다른 플랫폼에서 예약했다.
다른 호실로 배정했다.

**설계한 게 아니다. 운영하다가 발견한 것이다.**

이것이 호텔식 룸 어사인먼트의 원점이 됐다.

### 핵심 발견 2: 삼삼엠투 구조적 한계
삼삼엠투/리브는 7박 예약 구조다.
일요일 입실 → 토요일 퇴실 → 토요일 밤 1박이 빈다.
수요일에 입실하면 앞뒤 3일씩 날아간다.
1달 기준 가동률 75%.

**해결: 빈틈을 에어비앤비 단기로 채웠다.**

### 핵심 발견 3: 평일삼투 + 주말비엔비 공식
에어비앤비 요일별 가격: 토=100, 금·일=80, 월=70, 화~목=50.
중단기 게스트 중 평일만 이용하는 사람을 발견했다 (직접 물어봄).
1박분 환불하고 주말 2박을 에어비앤비로 판매했다.

```
환불: -₩4만
비엔비 토+일: +₩18만
순이익: +₩14만
```

### 핵심 발견 4: 수익 공식
```
매출 = 숙소 × 객단가 × 가동률
       ↑늘리고  ↓낮추고   ↑채우고
```

- 숙소: 월 2채씩 → 5채씩 증가
- 객단가: 시장 평균보다 낮게 (비엔비 7만→3만, 삼투 30만→25만)
- 가동률: 평일삼투+주말비엔비로 극대화
- "고객은 조금만 저렴해도 들어온다" → 가격으로 가동률 방어

### 수익 구조 비교 (방 1개 기준)
```
삼투만:    월 75~100만 (가동률 75%)
비엔비만:  월 136만 (평일 불확실, 가동률 70%)
HIERO식:  월 160만 (평일=삼투 확실 + 주말=비엔비 확실)
```

### 공급 확보 방법
1. 내 숙소 (렌트-인): 보증금 大. 자본 한계
2. 전대 (서브리스): 보증금 500만 + 월세 60만
3. **무보증/선세**: 보증금 0 또는 3개월 선세 + 월세 70만 → 대부분 이 방식
4. 경매 낙찰: 별도 맥락
- 셋팅비: 채당 50~100만원 (침대, 가전)

### 1단계가 가능했던 이유
- 공급자(경쟁자)가 없었다 — 강동구 초기 시장 독점
- 청소를 몸으로 했다 — 비용 0
- 가격을 낮췄다 — 경쟁력 = 가격
- 가동률 공식을 발견했다

### 1단계 한계
24년 5월, 주말 청소 10건 이상 발생.
3명으로 최대 5건. 물리적 한계.
숙소는 늘어나는데 청소가 안 돌아갔다.

### 1단계 데이터 (DB 기준)
```
24-01:   3건,  1숙소, ₩456만  (삼투만)
24-02:   7건,  1숙소, ₩816만

24-04:  14건,  1숙소, ₩1,437만
24-05:  73건, 12숙소, ₩3,233만  (비엔비 시작)
```

### 숙소 품질
- 자본 부족으로 최소한만 꾸밈
- "사진과 똑같아요" (칭찬이 아님)
- 경쟁 숙소 대비 시설 부족
- 하지만 가격이 낮아서 팔렸다

## 2단계: 조직화 + 공격적 확장 (2024.5 ~ 2024.12)

### 외부 환경 변화 (24.5월)
1. 에어비앤비: 무허가 숙소 퇴출 선언 → 강동구 경쟁자 다수 사라짐
2. 삼삼엠투: 전세사기 여파로 급성장
3. HIERO: 삼투 우수공급자 선정, 미팅, 상위호스트 혜택

### 사람 (5명 체제)
- **김진우**: 전략, 예약(24시간 수동), 기획, 대외 확장
- **김진태**: 셋팅 전담 (주 1채), 관리비 5종 납부
- **김대현** (6월~): 청소 시스템 구축 (외주 아줌마팀 + 자체빨래방)
- **왕태경** (6월~): 비용 처리, 월세 납부, 부동산 계약 관리
- **박수빈**: 데이터 누적, 정산, 월간 3지표 브리핑

### 운영 시스템
- **리셉션**: 엑셀 간트차트. 김진우가 수동 입력, 김대현이 보고 청소 배정
- 월간 브리핑: 숙소/객단가/가동률 3지표
- 청소: 건당 ₩33,000 → 10월 용역계약 ₩500만/월 (300건 손익분기)

### 24년 10월: 마지막 불꽃
삼투 누적 1억 확인. 월매출 8천만. "역시나 나는 잘해!"
- 숙소 확 늘림
- 외부 투자 유치
- 청소 용역 ₩500만 계약
- 책 기획 (생산/소비/투자/분배 프레임)
- **공격 모드**

### 24년 10월: 책 기획 — "김삼투" 이야기
이 시점에 운영 경험을 체계화하려 했다.
```
1장 생산: 숙소를 만드는 것
2장 소비: 고객이 경험하는 것
3장 투자: 수익을 극대화하는 것
4장 분배: 수익을 나누는 것
```
김삼투 = 23년 11월 24일 첫 예약을 한 본인.
"집은 비용이 아니라 자본이 될 수 있다."

### 24년 11월: 성장 공식의 붕괴
10월까지 "숙소만 늘리면 된다"가 작동했다.
11월에 깨졌다.

- 수능 → 크리스마스 전 = 소비 위축
- 비엔비 예약 급감
- 이사철 끝 → 장기 수요 소멸
- 계절 전환 → 이동 수요 없음

**깨달음:**
```
숙소(공급): 내가 통제 가능 ✓
객단가: 시장이 결정 ✗
가동률: 시즌에 의존 ✗
→ 3개 다 통제 가능하다는 전제가 틀렸다
```

비용은 올라갔는데 매출이 꺾였다.
투자금과 매출을 분리하지 않아 실제 하락을 인지 못했다.
통장에 8천만이 들어왔지만, 그 중 일부는 투자금이었다.

### 24년 12월 3일: 계엄
외국인 수요 50% 감소. 12월 1~3주 바닥.
마지막 1주(크리스마스~신정)에 12월 매출의 35%를 집중 매출.
하루 40건 청소. 화이트크리스마스를 무서워했다.

### 24년 채널별 연간 데이터
```
비엔비: 968건(79%) — 매출 ₩1.94억(46%)
삼투:   222건(18%) — 매출 ₩1.95억(46%)
리브:    28건(2%)  — 매출 ₩0.26억(6%)
개인:     5건      — ₩0.06억 (24년 개인입금 대부분 누락)

= 비엔비와 삼투가 매출 46%씩 정확히 반반
= 건수는 비엔비 79%인데 매출은 반반
= 삼투 건당 가치가 4배 높다
```

### 시즌 패턴 (첫 해 학습)

## 3단계: 재정비 (2025.1 ~ 2025.3)

### 사람
- 오재관 합류 (1월, 외부에서 운영 시스템 관찰)
- 이학주 합류 (1~3월, 진태와 숙소 셋팅 보조, 아직 적극 아님)

### 운영 시스템 (여전히 수동)
김진우가 24시간 수동으로 비엔비/삼투/리브 앱에서 예약 확인
→ 리셉션 엑셀에 수동 입력
→ 김대현이 리셉션 보고 청소 배정
→ 전부 수동. 공유 대시보드 하나.

### 데이터
```
25-01: 484건, 61숙소(DB부풀림), ₩1.30억 | 비엔비40%+삼투37%+개인20%
25-02: 300건, 58숙소(DB부풀림), ₩1.24억 | 삼투 55%! 비엔비 34%
25-03: 473건, 68숙소(DB부풀림), ₩1.75억 | 개인입금 88건 폭발
```
(실제 운영 숙소는 ~50채. DB는 Hostex 소급입력+임포트로 부풀려짐)

### 2월: 바닥
기억 기준 매출 ₩5천만. 비엔비가 완전히 빠짐.
**대응: "비엔비 다 빼고, 가격 낮춰서 장박으로 다 팔아라"**
DB에서 삼투가 55%로 1위 = 이 전략의 결과.

### 3월: 반등
이사철 시작 + 개인입금 급증 (17건→88건).
비엔비 복귀 + 삼투 유지 + 개인 폭발 = 월매출 ₩1.75억(DB).

### 0225 — 두 번째 회의 (2026.02.25, 약 4시간)

**핵심 논의:**

## 인벤토리 풀: HIERO의 핵심 경쟁력

### 발견 과정
1. 같은 방을 여러 플랫폼에 올림 → 방 변경 배정 발견
2. 중단기 사이 빈틈 → 단기로 채움 발견
3. 고객 패턴 파악 → 평일 고객에게 1박 환불, 주말 비엔비 판매
4. 비수기에 단기 포기 → 전부 장기로 전환

### 호텔식 룸 어사인먼트
```
A방에 에어비앤비 1박 예약
→ A방에 삼삼엠투 30박 장기 들어옴
→ 에어비앤비 게스트를 B방으로 변경
→ 게스트는 모름 (동급 이상 보장)
→ A방 30박 매출 + B방 1박 매출
```

개별 호스트는 이게 불가능하다.
방이 1~2개면 풀 자체가 안 된다.
청소/체크인 재배치도 운영 OS 없이 불가능하다.

### 경쟁사와의 차이
```
삼삼엠투: "A방을 30일" 매칭 (여기서 끝)
HIERO: "30일 수요가 오면 방을 재배치해서 전체 매출을 최적화"
```

## MORO × HIERO

### 컨셉
> 방을 파는 게 아니라, 거리를 팔고 문화를 팔고 경험을 판다.

### 구조
```
MORO (프론트): 게스트가 동네를 선택 → 예약
HIERO (백엔드): 인벤토리 풀에서 최적 방 배정 → 운영
```

### MORO Score Stack (변유진이 구체화)
- Layer 1 — 포기불가 지수: 경사도, 의료, 야간안전, 층/반지하
- Layer 2 — 골목 바이브 지수: 빵집/카페 밀도, 공원, 벤치, 인디상권
- Layer 3 — 외부 연결 지수: 통근시간, 대중교통, 따릉이/러닝코스
