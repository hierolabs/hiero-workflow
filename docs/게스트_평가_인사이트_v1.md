## 2. 분석 로직 (재현 가능하도록 기록)

### 2-1. 응답시간 측정 로직

```
입력: conversations 테이블 → messages 테이블 (sent_at ASC 정렬)

각 대화에서:
  for i = 0 to len(msgs)-1:
    if msgs[i].sender_type != "guest": skip
    if msgs[i].content == "": skip

    # 필터 1: 호스트 자동메시지에 대한 단순 답변 제외
    if 이전 메시지가 호스트 AND 자동메시지 키워드 포함:
      if 게스트 메시지가 50자 미만 AND 단순답변 키워드 포함:
        skip (오탐 방지)

    # 필터 2: 이슈 키워드 없으면 제외
    if 이슈 키워드 매칭 없음: skip

    # 다음 호스트 응답 찾기
    for j = i+1 to len(msgs):
      if msgs[j].sender_type != "host": continue

      # 필터 3: AI/자동 메시지 제외
      if 호스트 메시지에 "ai","자동","automated","auto","bot" 포함: skip

      # 필터 4: 야간(22~10시 KST) 30초 이내 응답 = 자동 판단
      if KST 22~10시 AND 응답시간 < 30초: skip

      # 유효 응답 → 기록
      response_time = msgs[j].sent_at - msgs[i].sent_at
      break
```

### 2-2. 필터 키워드 목록

**호스트 자동메시지 키워드 (제외 대상):**
```
체크인, check-in, 입실, 체크아웃, check-out, 퇴실,
감사합니다, thank you, 리뷰, review, 후기,
이용해주셔서, 즐거운, 편안한, welcome, enjoy,
ai, 자동, automated, auto, bot,
비밀번호, password, door code, wifi, 주소, address
```

**게스트 단순답변 키워드 (제외 대상):**
```
감사, 고마워, 고맙습니다, thank, thanks,
네, 넵, 넹, 알겠, ok, okay,
잘 쉬었, 좋았, 만족, 최고, 편했,
수고, 잘 지냈, good, great, nice, perfect,
ㅎㅎ, ㅋㅋ, ^^, :), 👍
```

**이슈 카테고리 키워드:**
```
boiler:      온수, 보일러, 에러, 난방, 가스, 뜨거운물, 차가운물
checkin:     안열, 안돼, 못들어, 잠겨, 잠김, 문이안, 번호가안, 출입이안
parking:     주차등록, 출차, 주차비, 차가안나, 주차가안
cleaning:    청소, 머리카락, 냄새, 수건, 이불, 더러, 벌레, 바퀴
reservation: 연장, 환불, 취소, 입금, 결제, 위약금
emergency:   화재, 가스누출, 물새, 정전, 119, 112
```

**안내 vs 조치 판별:**
```
조치 키워드: 수리, 교체, 출동, 기사, 방문, 설치, 보충, 배송
→ 호스트 응답에 조치 키워드 있으면 "action"(현장팀)
→ 없으면 "guide"(사무실팀)
```

### 2-3. 얼리/레이트 체크인·아웃 감지 로직

```
얼리 체크인:
  ("일찍" AND ("들어"|"입실"|"체크인")) OR
  ("얼리" AND "체크인") OR
  ("early" AND "check") OR
  ("미리" AND "들어") OR
  ("오전" AND "입실")

레이트 체크아웃:
  ("늦게" AND ("나가"|"퇴실"|"체크아웃")) OR
  ("레이트" AND "체크아웃") OR
  ("late" AND "check") OR
  ("더 있" AND ("가능"|"될까")) OR
  ("체크아웃" AND ("늦"|"연장"))

얼리 체크아웃:
  ("일찍" AND ("나가"|"퇴실"|"체크아웃")) OR
  ("먼저" AND "나가")

레이트 체크인:
  ("늦게" AND ("도착"|"들어"|"입실")) OR
  ("밤" AND "도착") OR ("새벽" AND "도착")
```

### 2-4. 주차 수요 감지 로직

```
주차 메시지 = content에 "주차" OR "parking" OR "차량" OR "차를" 포함

분류:
  가능 여부 문의: "주차" AND ("가능"|"되나"|"있나"|"할 수")
  등록/출차 도움: "등록" OR "출차" OR "주차비" OR "정기권"
  불가 언급:     "주차" AND ("안되"|"불가"|"없")
  정보 문의:     나머지
```

## 5. 운영 개선 액션

### 즉시 (시스템 변경 없이)
1. 10~12시 메시지 알림 강화 — 통화 중에도 메시지 체크 루틴
2. 보일러/체크인 → CS Knowledge 템플릿 1클릭 응답
3. 주차 안내를 체크인 메시지에 기본 포함

### 단기 (1~2주)
1. Hostex 과거 대화 전체 동기화 (backfill)
2. 30분 미응답 시 자동 에스컬레이션 알림
3. 얼리/레이트 요청 시 청소 스케줄 자동 확인

### 중기 (1~2개월)
1. 삼투/개인입금 메시지를 시스템 안으로 가져오기
2. 통화 STT → 자동 분류
3. AI 자동 초안 → 10~17시 메시지 무응답 방지
